CSS:

CSS- Cascading Style Sheets.
its like an organizer that describes how HTML elements will be displayed.
it determines fonts, colors, margins, lines, height, width, backgorund images and many other styling methods.

Syntax:

selector { property1:value1; property2:value2; }

Selectors:

tag names, class names (.className) and id (#id) are considered as selectors in css that can be used for selecting a particular section for styling
* is considered as universal selector
other selectors include psuedo class (based on certain state), psuedo elements (style a part of an element) and attribute selectors (select elements based on an attribute or attribute value).

Ways of inserting a CSS

  • Inline CSS -- inside a html element
  • Internal/Embedded CSS -- in the head section of html doc
  • External CSS -- done in a css file and is linked to the html doc
An inline style has the highest priority and will override internal and external style sheets.

BOX Model

It is a box that wraps around every html element.
It consists of:

  • margin -- area outside the border
  • border -- a border line that goes around the padding and content
  • padding -- area around the content and inside border line
  • content -- content of the box that has text, images or any kind of data
outline -- it is drawn outside the border, to make the elements stand out.
outline properties include:
  • outline-style
  • outline-color
  • outline-width
  • outline-offset -- adds space btw outline and border
  • outline
the outline and border styles include: dotted(.....), dashed(-----), solid(____), double(double outline), groove(3d grooved outline), ridge(3d ridged), inset(3d), outset(3d), none, hidden.

border:
border-radius: x px -- gives curved edges
border-style: dotted/dashed/double/ etc.
border-width: thick/medium/x px/top-bottom sides/ x px top, y px right/ z px left etc.
border-color: color/#num (hex value - 3/6 digit) rgb(x,y,z)/hsl(x%,y%,z%)/transparent
box-shadow: right/left top/bottom blurring color;

Layouts:

display proerties:

none
inline -- displayes element as inline. any ht and width will have no effect
block -- starts in a new line and takes up entire width
flex -- as block-level flex, 1D layout, sets the flexible length on flexible items by adjusting (increasing/decreasing) its size
flex-direction: reverse/row-reverse/coloumn-reverse
grid -- block-level grid container, 2D layout
inline-block - inline-level block and width and height can be applied
inline-flex -- inline-level flex
inline-grid -- inline-level-grid

positioning:

stactic -- not affected by top/bottom/right/left, it always stays acc to the flow of the page relative -- it will adjust itself away from normal position absolute -- will be positioned relative to the nearest positioned ancestor fixed -- the position will be fixed even if the page is scrolled

Z-index:

it sets the stack order of overlapping elements. if 2 positioned elements overlap without z index the last element will be shown
the element with higher z-index will appear in front of an element with lower z index, as it sets the order of an element

Float:

right/left/none
places an element on the left or right side of the container and allows the content to wrap around it

Clear:

none/right/left/both
the way element interact with floated elements is controoled by clear property. It shows if an element can be placed next to float elements

Responsive Design:

it resizes, hides, shrinks, enlarges or move the contents to fit all the devices (desktops, tablets, phones) veiwing the web page. it uses only html and css
viewport: user's visible area of a web page which is different for each device
<meta name="viewport" content="width=device-width, initial-scale=1.0">
name="viewport" -- controls the page's dimensions and scaling
width=device-width -- sets the width of the page to the screen width of the device
initial-scale=1.0 -- initial zoom level when the page is loaded

Media queries:

it should either be written in increasing (min-width: x px) or decreasing order (max-width: x px) and follows by {} with the properties written inside {}
the properties that have to be changed with the device width should be mention within media queries
eg: @media only screen and (max-width: 1200px) { selector { property:value; } }

Advanced Selectors:

Adjacent Sibling selector:

selects all elements that are adjacent siblings of the specified elements.
eg: h2+ul selects all ul next to h2

Attribute Selectors:

selects a particular type of input.
input[type="radio"] {property:value}

nth-of-type Selector:

selects an element from its position and types
element:nth-of-type(position/type){property:value;}

Direct Child Element:

seclect any element matching the second element that is a child of the first element.
childElement > parentElement {property:value;}

General Sibling Selector:

it selects the child element inside the parent element
parentElement ~ childElement

Descendant Selector:

It makes changes to the elements inside the other elements only
element1 element2 element3 {property:value;} -- it changes the content only in element3

Psuedo Classes:

used to define a special state of an element.
when user moves mouse over it
when an element gets focus
selector:link {p:v} -- for unvisited links
selector:visited {p:v;} -- for visited links
selector:hover {p:v} -- mouse over it
selector:active {p:v} -- selected
selector:focus{p:v} -- when it gets focus
selector:first-child {p:v} - first child pseudo class element
selector i:first-child {p:v} -- for each/particular child run as loop
selector:first-child i {p:v} -- for all childs selector:lang(language) {p:v} -- select element with lang attribute value staring with language
:nth-child(n) :nth-last-child(n) :root -- selects the document root element

Pseudo Elements:

it is used to style or insert a specified element ::after -- inserts content after every selector specified
::before -- inserts content before every sp selector
::first-letter -- selects every first letter
::first-line -- selects every first line
::selection -- selects the portion of an element that is selected by user

Grid Layout:

Container properties:

grid based layout with rows and columns without using float and positioning.
display:grid/inline-grid
vertical lines of grid -- grid columns
hrizontal lines of grid -- grid rows
space btw each row and col -- grid gap (column-gap/row-gap/gap)
lines btw cols -- col lines
lines btw rows -- row lines
for a grid to start at col line 1 upto col line 3 -- it should be written as:
selector { grid-column-start:1; grid-column-end:3;}
grid-template: -- a shorthand property for grid-template-rows(size of rows),columns(size and . of cols) and grid-areas (to display rows n cols)
justify-content:space-evenly/space-around-space-between/center/start/end
align-content:space-evenly/space-around-space-between/center/start/end
grid-column/row: col-row no / span num (num=no of rows/cols to be spaned)
naming grid areas - selector {grid-area: gridName}
grid-row/col-start/end - grid-area: row-start / column-start / row-end / column-end
grid-area: 1 / 3 / 4 / 5 -- the item will start at 1st row 3rd col and ends on 4th row 5th col

Transitions:

chane in property over a given duration.
it includes
property to add and duration of effect
transition-delay: 2s delay in starting the transition
transition-duration: total duration of the transition
transition-property: width/ height/background/any other property
transition-timing-function:ease(slow start, then fast, then end slow - default func)/ linear(same speed)/ease-in(slow start)/ease-in-out(slow start n slow end)/cubic-beizer(can define own values of the func) -- specifies the speed curve of the transition effect

Animations:

An animation lets an element change from one style to another.
to use css animations, keyframes should be specified as it holds the style of the element at specified times.
selector { animation: animationName transition}

@keyframes animationName { from{p:v} to{p:v} }
or
@keyframes animationName { 0% {p:v}
50% {p:v} ...
}
animation-iteration-count: num -- no. of times the animation should run, this should be declared within Selector

animation-direction: normal (forward)/reverse (backwards)/alternate (first plays forward then backwards) /alternate-reverse (first plays backwards then plays forward)

animation-fill-mode:none(animation begins with the default value)/ forwards(element will retain the style values that is set by last keyframes)/ backwards(the animation will be performed by the first keyframes value)/both (extends the properties in both directions) is used to set the element before keyframes is performed from the previously performed keyframes action

CSS Variables- custom properties

css variables can have global or local scope
:root - global variable that matches the document's root element.
before accessing the var() the variable have to be declared with --primary-name
:root {--black : #000; --light-gray: #xyz;}
selector {color: var(--black);}
makes the code easier to read and also easier to change the color values

calc()

used to perform calculation based on css property
this func accepts single parameter expression and performs mathematical expressions with operators + - * and /
eg: calc(30% - 40px) -- 30% width of parent width(100%) and -40px is the constant value given for providing space/gap from the next item/element/border
flex:1 1 calc(x% - y px) -- 1 -- flex grow - how much available space should an element take within a container, 1 -- flex basis - for initial length of the item, these 1 and 1 are default values

Minification:

It reduces the size of the CSS files so that they load faster.
it eliminates the unnecessary chars and spaces without impacting how browser interpret it.
